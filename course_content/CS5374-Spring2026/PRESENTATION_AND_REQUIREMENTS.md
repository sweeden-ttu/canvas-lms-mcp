# CS5374 Spring 2026: Presentation and Project Requirements

Extracted from course materials (syllabus PDF, Canvas assignments).

---

## Project Overview

- **Scope**: Team-based project related to software testing and program analysis
- **Constraint**: Projects must be defined within the scope of **"Trustworthy AI"**
- **Instructor**: Akbar S. Namin (akbar.namin@ttu.edu)

---

## Project Proposal (First Deliverable)

**Assignment**: "Team-based project proposal: Extended to Due Feb 2"  
**Due**: February 3, 2026 (5:30 AM)  
**Format**: One page proposal

### Required Sections

1. **Project Title**
2. **Project Personnel**
3. **Executive Summary**: up to 10 lines
4. **Deliverable (First Round)**: 5 lines
5. **Deliverable (Final/Second Round)**: 5 lines

### Notes

- Only one submission per group
- No need for each teammate to submit the proposal
- Projects must be within scope of "Trustworthy AI"

## Course Topics (Relevant to Project)

From syllabus outline:

- Software V&V, testing, adequacy criteria
- Black box, white box, model-based, graph-based testing
- Fault localization, mutation testing
- Model checking and formal verification
- AI/ML-based software testing
- LLM evaluation and debugging
- LangSmith + hands-on experience
- AI/LLM/RL evaluation
